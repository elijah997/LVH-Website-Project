# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Official website for Lake Victoria Hotel, Entebbe, Uganda. Static multi-page luxury hotel site — no build system, no framework, no package manager — pure HTML, CSS, and vanilla JavaScript. Open any `.html` file directly in a browser to preview.

## Architecture

### File Layout

- **7 HTML pages:** `index.html`, `about.html`, `offers.html`, `bookings.html`, `dining.html`, `gallery.html`, `contact.html`
- **1 CSS file:** `css/style.css` (~2400 lines) — all styles for all pages
- **1 JS file:** `js/main.js` (~440 lines) — all interactivity for all pages
- **Admin Dashboard:** `dashboard.html` + `css/dashboard.css` + `js/dashboard.js` — standalone SPA admin panel (mock data, Canvas charts, localStorage persistence)

### Shared Boilerplate (duplicated in every HTML page)

Every page manually includes the same blocks in this order: preloader (`#preloader`), scroll-progress bar (`#scroll-progress`), top-bar, header/nav (`#header`), and at the bottom: back-to-top button (`#backToTop`), mobile book bar (`#mobileBookBar`), footer. When editing nav links, footer content, or any shared block, the change must be replicated across all 7 HTML files.

### Single JS File, Multi-Page Aware

`main.js` queries all page-specific elements by ID at the top level and null-checks before use, so it runs without errors on every page. Page-specific features:
- **index.html only:** Hero slider (`.slide`, auto-advance 5.5s), floating booking widget with date picker defaults
- **gallery.html only:** Filter tabs (`.gf-btn[data-filter]`) and lightbox (`#lightbox`) — lightbox uses `getVisibleImages()` to respect active filter
- **contact.html only:** Contact form (`#contactForm`) with inline success message (`#cfSuccess`)
- **All pages:** Preloader, scroll progress bar, header scroll effect (`.scrolled` at 60px), mobile drawer nav, animated counters, back-to-top with SVG ring progress, scroll reveal, smooth anchor scrolling, newsletter/availability form interception

### Design System (CSS Custom Properties in `:root`)

- **Fonts:** `--font-heading` (Playfair Display), `--font-sub` (Cormorant Garamond), `--font-body` (Lato)
- **Core colors:** `--green: #2c6e49`, `--gold: #c9a84c`, `--cream: #f5f0e8`, `--green-dark: #1e4d33`, `--dark: #1a1a1a`
- **Shadows:** `--shadow`, `--shadow-lg`
- **Transition:** `--transition: all 0.3s ease`

### Key JS Patterns

- **Scroll reveal:** JS injects `.rv` class onto target elements and a `<style>` block. `IntersectionObserver` adds `.rv-in` to trigger fade-up animation with staggered delays per sibling index.
- **Animated counters:** Elements with class `.counter` and `data-target="N"` are animated to their target value using `requestAnimationFrame` with ease-out cubic easing, triggered by `IntersectionObserver`.
- **Mobile nav:** Right-side slide-in drawer (`#mainNav.open`), dark overlay (`#navOverlay.active`), body scroll lock. On mobile, `.has-dropdown` parent links toggle `.open` to expand submenus instead of navigating.
- **Form handling:** All forms use `preventDefault` and show a temporary "Sent!" state on the submit button. No backend — purely client-side feedback.

### External Dependencies (CDN only)

- Google Fonts: Playfair Display, Cormorant Garamond, Lato
- Font Awesome 6.4.0 (cdnjs)
- JSZip 3.10.1 (cdnjs, dashboard only — full website backup)
- All images: Unsplash URLs with `w=` and `q=` params for sizing/quality

### Responsive Breakpoints

CSS uses `max-width` media queries at: `1200px`, `992px`, `1024px`, `768px`, `576px`, `480px` (two responsive sections — general + multi-page components). Mobile sticky book bar hides via `IntersectionObserver` when footer enters viewport.

## Continuous Improvement Protocol

**You are expected to actively maintain and evolve this codebase.** After completing any task — bug fix, feature addition, refactoring, or styling change — follow these steps:

1. **Log what you changed** in the "Changelog" section below. One line per entry: date, what changed, and which files were touched.
2. **If you encountered a problem and solved it**, add it to the "Challenges & Solutions" section. Future instances will thank you — the same issue is likely to surface again.
3. **If you notice something that should be improved but is out of scope for the current task**, add it to the "Improvement Opportunities" section. Don't let good ideas get lost.
4. **If you discovered a gotcha, quirk, or non-obvious behaviour** in this codebase, document it in "Challenges & Solutions" so the next instance doesn't waste time rediscovering it.

Keep entries concise. One to two sentences max. This file is read at the start of every session — brevity matters.

## Challenges & Solutions

<!-- When you hit a problem and solve it, record it here so future sessions don't repeat the struggle. -->

| Challenge | Solution |
|-----------|----------|
| Shared markup (nav, footer, preloader, back-to-top, mobile bar) is copy-pasted across all 7 HTML files. Editing one and forgetting the others causes inconsistencies. | Always search-and-replace across all `.html` files when modifying shared blocks. Use grep/glob to verify all 7 files match after the change. |
| `main.js` runs on every page but many DOM elements only exist on one page. Adding a new feature that queries a missing element throws errors on other pages. | Always null-check with `&&` or `if (el)` before attaching listeners or manipulating page-specific elements. Follow the existing pattern in `main.js`. |
| `css/style.css` is a single ~2400-line file. Finding the right section to edit can be slow. | The file is organized with block comment headers (`/* --- Section Name --- */`). Search for the section name rather than scrolling. |
| On sub-pages (`about.html`, `offers.html`, etc.), the nav "Book Now" button markup appeared malformed — icon and closing `</a>` were on the same line as `navOverlay` with literal `n` instead of newlines. | **Fixed 2026-02-28.** Replaced the corrupted single line with the correct multi-line `nav-book-btn` anchor from `index.html`. |
| Gallery lightbox `lightboxGo()` crashed with modulo-zero when no images were visible after filtering. | **Fixed 2026-02-28.** Added `if (images.length === 0) return;` guard at top of `lightboxGo()`. |
| Slider `goTo()` crashed on sub-pages with no `.dot` elements (dots NodeList empty). | **Fixed 2026-02-28.** Added `dots[currentSlide] &&` null-checks before `classList.remove/add('active')`. |
| Contact page said "5 km from airport" while all other pages said "3.1 km". | **Fixed 2026-02-28.** Standardized to 3.1 km on `contact.html` (aside text + list item). |
| Generic form interceptor (`form:not(#contactForm)`) catches any new form added to any page, hijacking its submit handler. | When adding a new form with its own submit logic, exclude it from the generic selector: `form:not(#contactForm):not(#newFormId)`. |
| Dashboard changes were lost when localStorage cleared (cache clear, new browser, incognito). "Save to Site" only downloaded a file — user had to manually replace `js/site-config.js`, which they often didn't do. | **Fixed 2026-03-04.** "Save to Site" now uses `window.showSaveFilePicker` (File System Access API) to write directly to the file on disk. File handle is persisted in IndexedDB so future saves are one-click. Falls back to download in unsupported browsers. |

## Improvement Opportunities

<!-- Spotted something that could be better but it's not your current task? Drop it here for a future session. -->

- **Templatize shared HTML blocks:** Header, footer, preloader, back-to-top, and mobile book bar are duplicated verbatim across 7 files. Consider a simple include system (e.g., an HTML build step, or web components) to eliminate this duplication.
- **Split `style.css`:** The single CSS file is large. Could benefit from being split into partials (base, layout, components, pages) with a simple concatenation step.
- **Accessibility audit:** Add skip-to-content link, ensure all interactive elements have focus styles, verify ARIA roles on the lightbox and mobile nav.
- **Local image fallbacks:** All images are Unsplash CDN URLs. If the site is used offline or Unsplash is down, nothing loads. Consider downloading key hero/room images locally.
